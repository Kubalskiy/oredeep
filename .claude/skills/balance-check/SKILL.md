Домен из `$ARGUMENTS[0]`; если аргумента нет — `all`.

## Фаза 1. Измерить, а не угадывать

Всегда начинай с прогонов. Число из прогона бьёт любое рассуждение.

```bash
node tools/run_tests.js        # ждём 0 FAIL
node tools/calibrate_atk.js    # треадмилл: сек/жилу по глубинам + лог-сигма
```

Если нужна гипотеза, которую эти двое не покрывают, — пиши временный пробник:
достань `<script>` из `index.html`, склей с `tools/test_stub.js`, посчитай.
Готовый шаблон склейки — в шапке `tools/calibrate_atk.js`.

## Фаза 2. Прочитать источник правды

`index.html`: `BALANCE`, `DEPTH`, `ATK_COMPOUND`, `UPGRADES`, `STAT_CAPS`,
`ANCHOR_HP/ABR/HARD/RESP`, `MINE_DURAB`.
Контекст и «точные» значения из байткода — `docs/REVERSE_ECONOMY.md`.

## Фаза 3. Проверки по доменам

### treadmill — бесконечная сложность

- Равновесие держится: `escPerBlock() == ATK_COMPOUND ^ atkLevelsPerBlock()`.
- Лог-сигма сек/жилу на глубинах ≥3000 близка к нулю (было достигнуто **0.022**).
  Растёт — треадмилл кривится, ищи некомпенсированный множитель.
- Вкладывающийся игрок держится в коридоре ~6 сек/жилу; забивший отстаёт
  на порядки. Если забивший не страдает — статы стали не нужны.
- Разгон плавный: без ямы и без пика выше ~10 сек на `escStart..escStart+rampBlocks`.
- Что легко сломать: `resp` попал под эскалацию; `gearComp` убран; якорная
  кривая разморожена после `escStart`.

### upgrades — кривые апгрейдов

Для каждого из 8 `UPGRADES` посчитай **золото за единицу стата** на уровнях
0 / 20 / 50 / 100 / 200.

- Плоский `step` при `g > 1` = мёртвый апгрейд в лейте. Это уже случилось с ATK
  (8 → 971.79Sp золота за 1 ATK) и лечилось компаундом.
- Стат с потолком (`STAT_CAPS`) должен блокировать покупку на капе, а не жечь
  золото впустую.
- Каждый стат обязан где-то **читаться**. Проверь grep'ом: `tough` перестал
  применяться, когда LUCK на 100% начал полностью гасить трату энергии.

### economy — источники и стоки

- У каждой валюты (золото, кристаллы, осколки, пиво, кубки, ключи) — хотя бы
  один источник и хотя бы один сток.
- Стоки масштабируются вместе с доходом. Фиксированная цена при экспоненциальном
  доходе мгновенно обесценивается (`reinforceCost` привязан к `idlePerSec` — так и надо).
- Обслуживание крепи — лёгкий налог (~3.5% дохода), а не грайнд.
- Нет бесконечных петель: ресурс не должен порождать сам себя с коэффициентом ≥1.

### loot — дроп и лутбоксы

- Веса `bagWeights(1..50)` нормированы к 100.
- Pity-кривая геологов монотонна.
- Вклад камней в доход держится в коридоре 10–25%.
- Крафт и апгрейд боксов тратят **только дубликаты**; последний камень каждой
  редкости не расходуется.
- Ни один тир лута не бесполезен полностью на своей глубине.


## Фаза 5. После правки

Обязательно: `node tools/calibrate_atk.js` и `node tools/run_tests.js` заново.
Новый инвариант — новый тест в `tools/test_cases.js`. Без теста правка
проживёт до следующего коммита.
