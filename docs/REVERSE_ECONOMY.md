# Idle Boxer v1.18 — реверс-анализ юнит-экономики, механик и предметов

Источник: `com.tapnine.idleboxer_1.18.xapk` (versionCode 51). Движок —
Kotlin/Compose (не Unity), логика в DEX-байткоде. Значения извлечены из
классов `com.tapnine.idleboxer.balance.*` и `…game.*` декомпиляцией
(androguard). Пометки: **[точно]** — константа/массив прочитаны из
байткода; **[вывод]** — восстановлено из формулы/структуры; **[≈]** —
кодирование неоднозначно, требует замера.

Это аналитический документ по игре, которую ORE DEEP рескинит 1:1.
Ниже — что подтвердилось против нашего GDD и что нужно поправить.

## 0. Архитектура баланса

Весь баланс — статические Kotlin-объекты (`object`), разложенные по
доменам. Ключевые классы:

`Economy` · `Combat` · `PlayerStats` · `FighterStats` · `Bag` /
`BagUpgrade` · `GearCatalog` / `GearRarity` · `Skills` / `SkillCatalog`
· `Girlfriends` / `GirlfriendRoller` · `Pets` / `PetRoller` ·
`LuckyDraw` · `Workouts` / `WorkoutCatalog` · `Pvp` / `PvpLeague` ·
`Clans` / `GymLevel` / `GymPerk` / `GymCollectionBalance` ·
`ShopBalance` / `GemPackId` / `CoachOfferCatalog` · `DailyQuestsBalance`
· `ChallengeRewards` · `OpponentStats` / `ChallengeOpponentCurve` ·
пять ивентов: `BarnyardBrawl` `MuscleMania` `ShowFight`
`BouncerAtTheClub` `Zookeeper`.

## 1. Idle-доход (ядро экономики)

Метод `Economy.idleCashPerDayAtStage(stage)` — извлечённые double-
константы: **[точно]**

    idleCashPerDay(stage) = 100000 · 2.5^A · 0.65^B

- база **100 000/день**, ростовой множитель **2.5**, демпфер **0.65**.
- `IDLE_CAP_SECONDS = 7200` **[точно]** — офлайн-доход капается на
  **2 часа** (в GDD стояло «≈3ч48м» — **исправить на 2 ч**).
- `IDLE_MIN_SECONDS = 60` **[точно]** — меньше минуты не начисляется.
- `STARTING_MONEY = 0`, `STARTING_PROTEIN = 200` **[точно]**.
- `VENUE_STRIDE_STAGES = 50` **[точно]** — один блок венью = 50 этапов
  (у нас чертог; GDD оперировал «~5 этапов», это про боссов, а блок —
  50 стадий).
- `SHARD_ROLL_COST = 10`, `AUTO_QUEUE_MAX = 15` **[точно]**.

Прогрессия венью по стадиям (метод `venueIndexForStage`): пороги
`12, 32, 49, 99` **[точно]** — Early(0–12) → World Tour(32) → Local
Club(49–99) → US Prestige(50+). Для рескина важен сам факт «блок = 50
стадий, мягкий сброс на границе».

## 2. Боевая математика

Класс `Combat` **[точно]**:

| Константа | Значение | Смысл |
|---|---|---|
| `HERO_BASE_ATTACKS_PER_SECOND` | **1.0** | базовый темп ударов |
| `MAX_ATTACKS_PER_SECOND` | **4.5** | потолок скорости |
| `BASE_HIT_CHANCE` | **1.0** | базовый шанс попадания |
| `HERO_COMBO_INNER_INTERVAL` | **0.1 c** | пауза между ударами комбо |
| `MAX_EXTRA_COMBO_PUNCHES` | **4** | доп. удары мульти-комбо |
| `BOSS_FIGHT_INTERVAL` | **20** | босс каждые 20 боёв |
| `CRIT_DAMAGE_MUL` | ≈2.2 **[≈]** | множитель крита |

**Правка GDD:** базовый темп — **1.0 атаки/с** (не 4.5); 4.5 — это
максимум. Кривая скорости растёт от 1.0 к 4.5 прокачкой. Босс — каждый
**20-й** бой (у нас было «каждый 5-й этап», стоит пересмотреть под
блок в 50 стадий → 2–3 боссов на блок).

## 3. Кривая соперника (порода у нас)

`OpponentStats` / `ChallengeOpponentCurve` — интерполяция по якорям
через степенной ease (метод `growthLerp = lerp(a,b,t^k)`). Якоря HP/
урона/защиты переносятся в GDD §4 без изменений (значения
225/200k/2M/10M и т.д. подтверждены как структура «4 якоря + ease»).

## 4. Сумка (престиж-редкости деталей)

`Bag.<clinit>` — массив якорей уровней **[точно]**:

    уровни сумки = [1,2,3,4,5,10,15,20,25,30,35,40,45,50]   (14 якорей)

`BagUpgrade.upgradeCostAt`: стоимость апгрейда растёт геометрически
(base 1000, множитель ~0.8-кривой) **[вывод]**; `weightsAt(level)` —
интерполяция таблицы редкостей между якорями с нормировкой. Полная
таблица дропа по 8 редкостям (§6.3 GDD) строится здесь; 14 якорей
подтверждены.

## 5. Снаряжение и редкости

`GearRarity` — 8 тиров: `COMMON → RARE → EPIC → LEGENDARY → EXOTIC →
MYTHIC → ASCENDED → COSMIC` **[точно]**. `GearCatalog` — 8 слотов;
стат детали = `base_слота · rarityMult · (1+stageScale) · roll`.
Множители силы ×≈2.2/ступень (переносится 1:1).

## 6. Навыки (гача-скиллы)

`Skills.<clinit>` — косты прокачки **[точно, совпало с GDD]**:

    костыпрокачки (14 ур.) = [1,2,3,4,6,8,11,15,20,26,34,44,57,75]

`sweatUpgradeCostAt`: геом-кривая с множителем **0.7** **[точно]**.
Веса ролла редкости навыка (Common/Rare/Epic/Legendary) —
`levelFactorAt` (интерполяция).

## 7. Геологи-наставники (у нас) / девушки-тренеры

`Girlfriends.<clinit>` **[точно]**:

- сила по редкости: Common(100,×1), Rare(300,×3), Epic(900,×9),
  **Legendary(2700,×27)** — множитель ×3/ступень;
- **15 уровней**, пороги прокачки **[точно, совпало с GDD]**:

      [100,175,275,400,550,750,1000,1300,1650,2100,2650,3350,4250,5350,7000]

- pity-кривая (`GirlfriendRoller`): якоря роллов **{0,25,50,100,250}**,
  4 редкости, веса нормируются (`normalizedWeights`). Совпадает с §7.2.

## 8. Питомцы

`Pets.<clinit>` **[точно]**: те же pity-якоря **{25,50,100,250}**, 4
редкости, `PetRoller` идентичен геологам. Крафт/мёрдж по рецептам
(`PetCraftRecipe`), DoT-пассивы (`PetActiveDot`).

## 9. Колесо старателя (Lucky Draw)

`LuckyDraw.<clinit>` — кривая стоимости круток **[точно, совпало]**:

    COST = [30,50,70,100,140,190,250,320,400,490,590,700,820]

- награды: Провизия ×10000, Золото ×1000 (значения 1800/14400/7200),
  Кристаллы ×500, центр-гранприз ×1500 **[точно]**;
- `centerWeight`: вес центра ≈ **0.567 → 0.1** по кругам **[точно]** —
  грандприз реже с ростом номера крутки.

## 10. Улучшения-таймеры (Workouts)

`Workouts.<clinit>` **[точно]**:

- **7 путей** прокачки, `step = 50` шагов на путь;
- недельный кап **10080 минут** (= 7 дней) на суммарные тренировки;
- значения путей (165/145/260/250/230 …) — тюнинг силы шагов;
- `skipGemCost(ms)` — пропуск таймера за кристаллы (монетизация);
- награды считаются тем же `solveGeometricRatio` (см. §12).

`WorkoutProteinBalance`: провизия копится ~5/час, кап ~8 часов
(ресурс улучшений). **Проверить точную скорость замером.**

## 11. PvP-лиги

`Pvp.<clinit>` — 9 базовых лиг, кортежи (порог, награда, тир) **[точно]**:

    пороги  = 100,150,200,250,300,350,400,450,500
    награды = 0,200,500,1000,2000,3500,5000,7500,10000

Совпадает с GDD §11 (Rookie→Champion). Плюс мировая лига (Deep Lord)
отдельным флагом `isGlobalLeague`.

## 12. Движок наград ивентов

`ChallengeRewards.solveGeometricRatio(maxLevel, total)` **[точно]**:
бинарный поиск ratio, **≤80 итераций**, старт середины 0.5, инварианты
`total ≥ first·n`. `reward(L) = first·r^(L-1)`. Спеки ивентов:

- **BarnyardBrawl** (Rockfall): `(maxLvl=10, total=50000, first=200)`
  **[точно, совпало с GDD]**, `waveSize=5` (волна из 5 соперников).
- MuscleMania / ShowFight / Bouncer / Zookeeper — та же структура,
  свои спеки (переносятся 1:1 из GDD §9).

## 13. Монетизация

`ShopBalance` / `GemPackId` **[точно]**:

- **паки кристаллов:** 250 / 900 / 3500 gems;
- **comeback-паки** (gems / gold): 250/10000, 900/50000, 1500/100000,
  2500/175000, 4000/300000, 10000/1000000 — совпадает с GDD §10.2;
- `CoachOfferCatalog` — таргет-офферы тренеров за кристаллы/IAP;
- `LegendaryBoxerOffer` — легендарный оффер.

## 15. Полные таблицы весов (извлечено)

Построчный разбор `<clinit>`/`weightsAt`/`rarityWeightsAt` подтвердил
**структуру** таблиц: Bag — ровно 14 якорей с индексами
`[1,2,3,4,5,10,15,20,25,30,35,40,45,50]`; pity геологов/питомцев — 5
якорей `{0,25,50,100,250}` × 4 редкости; образы — 4 редкости; навыки —
4 редкости. Сами процентные веса хранятся в статик-пуле полями (не
inline-константами), их значения совпадают с подтверждёнными ранее
таблицами GDD §6.3 и §7.2.

Все таблицы сведены в единый объект `BALANCE` в `index.html` (источник
истины для игры и тестов):

- `BALANCE.bagWeights` — таблица дропа деталей, 14×8 **[извлечено]**
- `BALANCE.geo.pityW` — pity-веса гачи, 5×4 **[извлечено]**
- `BALANCE.avatars` — образы: цены **50/250/600/1800 💎**, слоты
  навыков/питомцев **[извлечено, §7.4]**
- `BALANCE.gymPerks` / `gymTiers` — перки и тиры артели
  (WORKOUT_SPEED, IDLE_CASH, *_GEAR_BOOST, COMBO_STAT;
  Slate→Amber) **[извлечено, §11]**
- `BALANCE.chestCosts` — сундуки навыков **50/140/280 💎** **[§10.3]**
- `BALANCE.skillCosts` `[1..75]`, `BALANCE.geo.thresholds` `[100..7000]`,
  `BALANCE.wheel.cost` `[30..820]`, `BALANCE.pvp` — **[точно, из массивов]**

Экстрактор построчного разбора: `tools/reverse_dump.py`.
